# 실행 가이드 (프리모르템 + 시너지맵 + 정량모델 + Gotchas)

## §5. 프리모르템 검증 (3모드 공통 종료 단계)

모든 모드의 마지막에 실행. 생략 금지.

```
전제: "이 [진단/설계/변환]이 실패한다면 왜?"
→ V1(이행 근거) 위반 가능성?
→ V2(정체성 침해) 위반 가능성?
→ V3(자극 과잉) 위반 가능성?
→ V3-t(자극 소진) 위반 가능성? (반감기·대응 전략 유무 — v3t-decay-model.md 참조)
→ V5(경계 부재) 위반 가능성?
→ L3 축C(시대정신 불일치) 가능성? 담론 수명 단계는? (온타임/앤타임/포스트타임 — domain-universal.md 횡단법칙 §4)
→ 기술 자극이 감정 설계를 대체하고 있지 않은가? (기술↑ ≠ 감정↑, 특히 B·F 도메인)
→ 치명 리스크 Top 3 + 각각의 사전 차단책

하이브리드 도메인인 경우 추가:
→ Host-Guest 역전 가능성? (guest가 host를 잠식)
→ V2 하향 수렴 위험? (낮은 쪽으로 끌려감)
→ L3 자원 분산 위험? (양쪽 L3 모두 중도반단)
→ V3-t 비동기 위험? (한쪽만 소진되어 하이브리드 붕괴)
→ E(전파 레이어) 오용 위험? (E를 파트너로 취급하여 V2 충돌)
```

## §6. 공식 간 시너지 맵 (조합 설계 참고)

| 조합 | 시너지 | 왜 강한가 |
|---|---|---|
| ①+③ | 잡음 제거 → 새 참조점 가속 | 제거가 새 기준의 선명도를 높임 |
| ③+④ | 참조점 전복 → 감정 반전 출발점 | 기존 기준 파괴의 부정 감정이 반전 재료 |
| ②+⑥ | 공백 → 반복의 변형 슬롯 | 수용자가 채우는 구조가 무한 반복 가능 |
| ④+⑤ | 감정 반전 피크 → 다중감각 각인 | 반전 순간의 감각 중첩이 기억 극대화 |
| ①+② | 잡음 제거 → 공백 생성 | 제거한 자리가 수용자 삽입 공간 |

## §7. 정량 모델 계산기 (선택)

L3 전파 수식을 `scripts/quant_models.py`로 코드화. LLM 인라인 계산 오류를 구조적으로 차단한다.

```bash
# ZM 포화 보정
python scripts/quant_models.py zm_eff 3.3

# NPE 산출 (PAM IRR ZM_raw DAC)
python scripts/quant_models.py npe 2.5 1.8 3.3 0.15

# 팬덤 가속 NPE (PAM IRR ZM_raw DAC OrgMob UGCamp CrossPlat)
python scripts/quant_models.py npe_fandom 2.5 1.8 1.5 0.15 3.0 2.0 2.5

# MCR 다채널 도달 (도달비중 비중복률 쌍)
python scripts/quant_models.py mcr 0.27 1.0 0.15 1.0 0.17 1.0

# 종합 산출
python scripts/quant_models.py all --pam 2.5 --irr 1.8 --zm 3.3 --dac 0.15 --fam_org 3.0 --fam_ugc 2.0 --fam_cross 2.5
```

**원칙**: 정량 모델이 필요한 경우(E전파 분석, ZM>2.0, 팬덤 가속) 이 스크립트를 우선 사용. 단순 비교·판정만 필요한 경우 인라인 판단 허용.

## Gotchas

| # | 안티패턴 | 올바른 접근 |
|---|---|---|
| 1 | 6공식 전부 적용 시도 | 2~3개 핵심 공식에 집중. 나머지는 보조 |
| 2 | V 체크 없이 공식 적용 | 공식 적용 전 V1~V5 반드시 선행 |
| 3 | L2만 보고 L1·L3 무시 | 3층 모두 스크리닝. L1(왜 반응?)이 L2의 전제 |
| 4 | "히트 공식 = 성공 보장" | 공식은 필요조건. 충분조건이 아님 (25개 반례 증명) |
| 5 | 시대정신을 설계하려 함 | 시대정신은 읽기만 가능. L3 축C는 게이트 |
| 6 | 스포크 전체 로드 | 스크리닝 후 해당 스포크만 로드. 토큰 절약 |
| 7 | 범용 도메인에서 사례 무비판 적용 | domain-universal.md의 조정 규칙 참고 |
| 8 | 부모 도메인 어댑터만 보고 세부도메인 무시 | 세부도메인에서 V2·L3 1순위가 부모와 달라지는 경우 있음 (§2.5) |
| 9 | 33개 세부도메인 어댑터 전체 로드 | §2.5에서 판별된 1개만 로드. 접경 지대는 판별 기준 참고 |
| 10 | 어댑터 없이 킬러 공식 균등 가중치 적용 | 반드시 도메인 클러스터(부재-반전/공백-반복/참조점-감각)에 따른 차등 가중치 |
| 11 | 하이브리드 도메인에서 E(숏폼)를 파트너로 취급 | E는 전파 가속 레이어로만 활용. V2 충돌 위험 (§2.5 Step 5, cross-domain-matrix.md) |
| 12 | E4(롱폼바이럴)를 E1·E3과 동일 취급 | E4는 V2=강, 킬러=①③으로 독립 콘텐츠 도메인. E의 "전파 레이어" 규칙 적용 불가 (subdomain-full-matrix.md 발견 1) |
| 13 | 6×6 부모 매트릭스만으로 하이브리드 설계 | 세부도메인 레벨에서 점수가 부모와 다를 수 있음. 세부도메인 특정 가능 시 subdomain-full-matrix.md 우선 참조 |
| 14 | 3개+ 도메인 하이브리드를 듀얼 2개로 분해하여 처리 | 트리플은 고유 시너지(삼각 공명+클러스터 교차)가 있음. triple-hybrid-matrix.md의 원칙 6~10 적용 필수 |
| 15 | 트리플에서 제3 도메인을 장식으로 취급 | 제3 도메인이 실질적 킬러 공식을 기여하지 않으면 가짜 트리플. "듀얼 + 마케팅 라벨"인지 검증 필수 |
| 16 | NPE·ZM 등 정량 모델을 LLM 인라인 계산 | scripts/quant_models.py 사용. 수식 오류 구조적 차단 |
