# §4.5 리포트 변환 — 상세 가이드

## 3층 구조

| 층 | 내용 | 비율 |
|---|---|---|
| **1층 원페이저** | 킬러 강점 1 + 치명 약점 1 + 최우선 액션 1. 3문장 이내 단언 | 상단 10% |
| **2층 서사 해설** | 강점·약점·처방·전파를 인과 문단으로. 축/공식명은 괄호 레이블만 | 중간 60% |
| **3층 데이터 접힘** | §2 매트릭스·V 체크·L3 5축 → 옵시디언 callout 접힘 (`> [!info]-`) | 하단 30% |

## 2층 서사 규칙

- 표 나열 ✗ → 문단 서술 ◯. 각 강점/약점 = 1문단
- 문단 구조: **입력물 구체 근거 인용 → 메커니즘 → 효과/리스크**
- 처방은 약점 문단 말미에 "→ 처방:" 자연 연결
- 프리모르템·V 경고는 별도 섹션 (서사+경고 혼합 방지)

## 모드별 2층 차이

| 모드 | 2층 초점 |
|---|---|
| 진단 | "왜 작동/안 작동하는가" |
| 설계 | "왜 이 공략인가 + 어떻게 실행하는가". 액션 플랜은 서사 후 별도 블록 |
| 변환 | "무엇을 왜 바꿨는가". Before/After는 2층 유지 (접힘 아님) |

## .md 템플릿

```markdown
# [대상명] 히트 [진단/설계/변환] 리포트

## 판정
> 킬러 강점 | 치명 약점 | 최우선 액션

## 왜 작동하는가
(강점 2~3개, 각 1문단 인과 서사)

## 왜 안 작동하는가
(약점 2~3개, 각 1문단 인과 서사 + → 처방:)

## 전파 가능성
(L3 핵심만 서사)

## 프리모르템
(리스크 Top 3 + V 위반 경고)

> [!info]- L1 스크리닝 매트릭스 (16축)
> (§2 테이블 원본)

> [!info]- L2 공식 + V 체크
> (6공식 + 5V 테이블 원본)

> [!info]- L3 전파 5축
> (5축 테이블 원본)
```
