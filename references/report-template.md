# §4.5 리포트 변환 — 상세 가이드

## 3층 구조

| 층 | 내용 | 비율 |
|---|---|---|
| **1층 원페이저** | 킬러 강점 1 + 치명 약점 1 + 최우선 액션 1. 3문장 이내 단언 | 상단 10% |
| **2층 서사 해설** | 강점·약점·처방·전파를 인과 문단으로. 축/공식명은 괄호 레이블만 | 중간 60% |
| **3층 데이터 접힘** | §2 매트릭스·V 체크·L3 5축 → 옵시디언 callout 접힘 (`> [!info]-`) | 하단 30% |

## 2층 서사 규칙

- 표 나열 ✗ → 문단 서술 ◯. 각 강점/약점 = 1문단
- 문단 구조: **입력물 구체 근거 인용 → 메커니즘 → 효과/리스크**
- 처방은 약점 문단 말미에 "→ 처방:" 자연 연결
- 프리모르템·V 경고는 별도 섹션 (서사+경고 혼합 방지)

## 모드별 2층 차이

| 모드 | 2층 초점 |
|---|---|
| 진단 | "왜 작동/안 작동하는가" |
| 설계 | "왜 이 공략인가 + 어떻게 실행하는가". 액션 플랜은 서사 후 별도 블록 |
| 변환 | "무엇을 왜 바꿨는가". Before/After는 2층 유지 (접힘 아님) |

## .md 템플릿 — 진단/설계 모드

```markdown
# [대상명] 히트 [진단/설계] 리포트

## 판정
> 킬러 강점 | 치명 약점 | 최우선 액션

## 왜 작동하는가
(강점 2~3개, 각 1문단 인과 서사)

## 왜 안 작동하는가
(약점 2~3개, 각 1문단 인과 서사 + → 처방:)

## 전파 가능성
(L3 핵심만 서사)

## 프리모르템
(리스크 Top 3 + V 위반 경고)

> [!info]- L1 스크리닝 매트릭스 (16축)
> (§2 테이블 원본)

> [!info]- L2 공식 + V 체크
> (6공식 + 5V 테이블 원본)

> [!info]- L3 전파 5축
> (5축 테이블 원본)
```

## .md 템플릿 — 변환 모드

**변환 모드는 이 템플릿을 정확히 따르라.** 2층에 "설계 원리", "변환 설명", "변환 이유" 같은 분석 해설 블록을 두지 마라. 그것들은 3층 접힘에 넣어라.

```markdown
# [대상명] 히트 변환 리포트

## 판정
> 킬러 강점 | 치명 약점 | 최우선 액션

## 왜 안 작동하는가
(원본 약점 2~3개, 각 1문단 서사. 처방은 "→ 처방:" 자연 연결)
(공식명·축명은 문장 끝 괄호 레이블만. ✗ "공식③을 적용" → ◯ "기준을 교체했다(공식③)")

## 변환 결과
### Before
(원본 텍스트)
### After
(변환 텍스트)
### 무엇이 왜 바뀌었는가
(각 변경 포인트를 수용자 관점 서사로. "이 부분을 이렇게 바꾼 이유는 수용자가 ~하기 때문이다(축N)")
(✗ "공식①을 적용하여 잡음을 제거" → ◯ "주의를 분산시키는 요소를 제거하여 핵심 메시지만 남겼다(공식①)")

## 전파 가능성
(L3 핵심만 서사)

## 프리모르템
(리스크 Top 3 + V 위반 경고)

> [!info]- L1 스크리닝 매트릭스 (16축)
> (§2 테이블 원본)

> [!info]- L2 공식 + V 체크
> (6공식 + 5V 테이블 원본)

> [!info]- L3 전파 5축
> (5축 테이블 원본)

> [!info]- 도메인 어댑터
> (도메인·세부도메인·킬러공식·V등급)

> [!info]- 변환 설계 원리 (분석 상세)
> (여기에 공식별 적용 상세, 변환 이유, 메커니즘 해설 배치)
```
