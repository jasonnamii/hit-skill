# Report Layer — 히트스킬 3-Layer 리포트 구조

히트스킬 분석 결과를 **의사결정자 관점 역순**으로 재구성하는 후처리 프레임.
분석 엔진(§1~§4)의 실행 순서와 리포트 출력 순서를 분리한다.

---

## 설계 원리

| 원리 | 내용 |
|------|------|
| **독자 역순** | 16축 매트릭스·6공식 상세가 먼저 나오면 "데이터 시트" 상태. 판정 문장이 첫 줄 |
| **판정 문장 필수** | 스크리닝 표만으로는 "그래서 뭘 어쩌라고?"에 답하지 못함. 강점·약점·리스크를 한 문장으로 종합 |
| **분석 과정 ≠ 산출물** | §2 스크리닝·§3 스포크 로딩·V체크는 분석 도구. 산출물은 Layer 구조로 재포장 |

---

## Layer 1: 판정 (Verdict)

**위치**: 리포트 최상단
**분량**: 3~5문장

### 판정 문장 생성 로직

1. §2 스크리닝에서 **강활성 축/공식 Top 3** 식별 (어댑터 가중치 반영)
2. **킬러 판정**: Top 3 중 해당 도메인에서 결정적인 것이 있는가?
3. **치명 약점**: V 위반, L3 취약점, 또는 킬러 축 미활성 중 가장 큰 리스크
4. 한 문장 종합: "[강점 요약]. 단, [치명 약점/V 위반]이 [구체적 리스크]를 만든다."
### 모드별 판정 문장 예시

| 모드 | 예시 |
|------|------|
| **진단** | "축2(손실회피) + 공식③(참조점 전복)이 강하게 작동 — 구매 전환의 킬러. 단, V1(이행근거) 부재로 재구매 시 신뢰 붕괴 리스크." |
| **설계** | "타겟의 축C(정체성) + L3-C(시대정신)을 동시 공략하면 자발적 전파 구조 가능. 단, V3(자극 과잉)으로 마비 리스크 — 자극 강도를 역U자 하단에 맞춰야." |
| **변환** | "원본은 축1(S1 경로)에만 의존 — 정보 전달은 되나 행동 유도 실패. 공식④(감정 반전) 삽입으로 축5(감정선행)를 활성화하면 CTA 전환율 상승." |

## Layer 2: 설계/변환안 (Core Deliverable)

**위치**: 판정 바로 다음
**분량**: 리포트 본문의 중심

### 모드별 L2 구성

| 모드 | L2 내용 |
|------|---------|
| **진단** | 처방카드 핵심 요약 (약점별 1-liner 처방 + 우선순위) |
| **설계** | 액션 플랜 (L1 공략 축 → L2 공식 조합 → L3 전파 전략 → 프리모르템 요약) |
| **변환** | Before/After 대조 + 리라이팅 결과물. 각 변환 포인트에서 "이 변경이 어떤 축/공식을 활성화하는가"를 논거로 녹임 |

**핵심**: §2 스크리닝 매트릭스를 독립 섹션으로 나열하지 않는다. 처방/설계/변환 논거 안에서 필요한 만큼만 인용.

예시 (변환 모드):
- ✗ "축5(감정선행)가 비활성입니다. 공식④를 적용합니다." → 데이터 시트
- ✓ "오프닝 3줄을 팩트 나열에서 감정 반전 구조로 전환합니다. 기존: '시장 규모 500억' → 제안: '당신이 매일 쓰는 서비스가 내일 사라진다면?' — 축5(감정선행)를 활성화하여 S1 경로로 주의 포착." → 논거가 녹아든 설계안
---

## Layer 3: 분석 근거 (Evidence Appendix)

**위치**: 리포트 하단 (접이식/부록)
**분량**: 필요한 만큼

| 포함 항목 | 출처 |
|-----------|------|
| 16축 스크리닝 매트릭스 (활성/비활성 + 근거 인용) | §2 L1 |
| 6공식 스캔 결과 + V 체크 상세 | §2 L2, L2-V |
| L3 5축 스캔 결과 | §2 L3 |
| 도메인 어댑터 세팅 + 가중치 | §2.5 |
| 처방카드 원문 (진단 시) | §4 |
| 프리모르템 상세 | §5 |
| 정량모델 계산 결과 (해당 시) | §7 |

### 출력 형식

| 컨텍스트 | 형식 |
|----------|------|
| 대화 응답 | 접이식 (details/summary 또는 콜아웃) |
| .md 파일 산출물 | `## 부록: 분석 근거` 섹션으로 하단 배치 |
| 옵시디언 | `> [!info]- 분석 근거` 접이식 콜아웃 |

## Gotchas

- **L1 스킵 금지**: 16축 매트릭스를 먼저 보여주고 판정을 뒤에 놓으면 다시 데이터 시트. 반드시 판정 문장이 첫 줄
- **L2에서 분석 도구 직접 노출 주의**: "§2 스크리닝에 따르면…" 대신 "구조 분석 시 X가 드러났다"처럼 논거로 녹여야
- **L3 생략 금지**: 근거 없는 판정은 신뢰를 잃음. 접이식이라도 반드시 포함
- **킬러 판정의 도메인 의존성**: 같은 축이 활성이어도 도메인에 따라 킬러인지 보조인지 달라짐. 어댑터 가중치 반영 필수
- **V 위반은 L1에 즉시 반영**: V 위반은 부록에만 넣으면 안 됨. 판정 문장에 "단, V[N] 위반으로 [리스크]" 형태로 반드시 포함
