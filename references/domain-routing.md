# 도메인 어댑터 라우팅 + 스포크 라우팅

## §2.5. 도메인 어댑터 라우팅

1차 스크리닝 직후, 입력물의 **도메인 → 세부도메인**을 판별하여 어댑터 파라미터를 세팅. 이후 §3~§5의 모든 분석·설계·변환은 어댑터가 세팅한 가중치를 전제로 실행.

### Step 1: 6대 도메인 판별

입력물의 소비 양식으로 1차 분류:

| 소비 양식 | 도메인 | 킬러 클러스터 |
|---------|--------|------------|
| 시간축 몰입 (순차·장시간) | **A 서사몰입** / **D 청각퍼포먼스** | 부재-반전계 |
| 행위 참여 (상호작용·루프) | **B 인터랙션몰입** / **E 숏폼바이럴** / **F 체험환경** | 공백-반복계 |
| 공간적 즉시 (한 순간·시각) | **C 시각즉시** | 참조점-감각계 |

**애매한 경우**: 형에게 도메인 확인. 복수 도메인 해당 시 주도메인 1개 선정 + 부도메인 참고.

### Step 2: 33개 세부도메인 판별

도메인 확정 후, 세부 매체·소비 맥락으로 2차 분류:

| 도메인 | 세부도메인 (33개) |
|--------|----------------|
| **A** | A1영화 · A2드라마 · A3애니 · A4웹툰 · A5출판 · A6팟캐스트 · **A7인터랙티브픽션** |
| **B** | B1게임 · B2UX/기능 · **B3소셜네트워크** · B4웹/추천 · B5라이드셰어 · B6XR |
| **C** | C1광고 · C2파인아트 · C3팝/디지털아트 · C4제품디자인 · C5패션 · C6브랜드CI · C7건축 |
| **D** | D1음악 · D2공연 · D3예능/방송 · **D4ASMR·사운드** |
| **E** | E1숏폼영상 · E2소셜콘텐츠 · E3밈 · **E4롱폼바이럴** |
| **F** | F1테마파크 · F2F&B/리테일 · F3페스티벌 · **F4호텔·숙박** · **F5설치미술·몰입전시** |

**접경 지대 판별 기준** (14건):

| 접경 | 판별 기준 |
|------|----------|
| 뮤직비디오 | 3분+ = D1, 1분- = E1, 중간 = 주소비채널 |
| 극장 애니 | 원작 연속성 有 = A3, 독립작 = A1 |
| 건축 체험 | 건물 감상 = C7, 체험 프로그램 중심 = F |
| 게임 VR | 기존 게임 VR이식 = B1, VR 네이티브 = B6 |
| 브랜드 광고 | 캠페인 단위 = C1, 영구 심볼 = C6 |
| 음악 챌린지 | 원곡 중심 = D1, 챌린지 참여 = E1 |
| 팝업매장 | 상시 운영 = F2, 기간 한정 이벤트 = F3 |
| 선택형 게임 | 분기 서사가 핵심 = A7, 게임플레이가 핵심 = B1 |
| 커뮤니티 앱 | 커뮤니티 소속감 = B3, 기능적 유틸리티 = B2 |
| 수면 음악 | 이완·수면 목적 = D4, 감상 목적 = D1 |
| 장편 교육 영상 | 영상 중심 = E4, 오디오 중심 = A6 |
| 디자인 호텔 | 숙박 체험 = F4, 건물 감상 = C7 |
| 몰입형 전시 | 상설/순회 전시 = F5, 기간 한정 축제 = F3 |
| 명상 앱 | 이완 콘텐츠 = D4, 기능 UX = B2 |

### Step 3: 어댑터 로드 + 파라미터 세팅

```
세부도메인 확정
  ↓
references/subdomain-adapters.md에서 해당 어댑터 로드
  ↓
4개 파라미터 세팅:
  ①킬러 공식 조합 + 가중치 (부모 대비 재조정값)
  ②V 조건 해석 (V2 등급 · V3 임계 · V5 경계)
  ③L3 축 계층 (세부도메인별 우선순위)
  ④감정경로 + 딜레이 모델
  ↓
세팅된 파라미터로 §3~§5 실행
```

**33개 밖 도메인**: 세부 어댑터 해당 없음 → `references/domain-universal.md`의 도메인별 조정 가이드(정책·투자자·교육·UX·개인브랜딩) 로드.

### Step 5: 하이브리드 도메인 판별 (2개+ 도메인 해당 시)

단일 도메인으로 분류 불가한 입력물(뮤직 페스티벌, 게임 원작 영화 등)이 들어오면:

```
①단일 도메인 판별 시도 → 실패 (2개 도메인 해당)
②references/cross-domain-matrix.md 로드 (부모 도메인 레벨)
 + references/subdomain-full-matrix.md 로드 (세부도메인 레벨 528쌍)
③매트릭스에서 해당 쌍의 등급 확인 (★★★/★★/★/⚠)
 → 세부도메인까지 특정 가능하면 전수 매트릭스 우선 참조
④5대 원칙 적용:
  원칙1: Host-Guest 결정 (V2 높은 쪽 = host)
  원칙2: 킬러 공식 선택 (host 2개 + guest 1개, 시너지맵 우선)
  원칙3: V2 상향 수렴 (높은 쪽 기준)
  원칙4: L3 전략 결정 (합류/시간분리/채널분리)
  원칙5: V3-t 동기화 (짧은 쪽 반감기 기준)
⑤⚠ 등급이면 하이브리드가 아닌 "host + 전파 채널" 구조 검토
⑥3개+ 도메인 해당 시 → references/triple-hybrid-matrix.md 추가 로드
  → 5,456 트리플 조합에서 등급 확인 (★★★★/★★★/★★/★/⚠)
  → 피라미드 Host 결정 (원칙 6~10 적용)
```

**E(숏폼) 관련 입력**: E는 하이브리드 "파트너"가 아닌 **전파 가속 레이어**. 다른 도메인의 콘텐츠가 E 형식으로 축약·전파되는 구조로 처리.
  → `references/e-propagation-model.md` 로드
  → NPE(X) 산출 → BFC(X) 확인 → 의사결정 플로우 적용
  → V3-t DAC 트레이드오프 고지 (E1: -23%, E4: -9%)
  → E4 포함 시 독립 도메인으로 하이브리드 판정 전환
  → **팬덤 재생산 활성 시** → `references/fandom-acceleration.md` 로드
    → FAM 파라미터 산출 (OrgMob·UGCamp·CrossPlat) → NPE_fandom 산출
    → 팬덤 성숙 단계별 전략 적용 (Ⅰ맹아/Ⅱ조직화/Ⅲ생태계)
  → **다채널 전략 필요 시** → `references/multi-channel-propagation.md` 로드
    → 도메인×채널 적합도 매트릭스 → CSE 산출 → 채널 조합 확정
  → **IRR 정밀 판정 필요 시** → `references/irr-calibration.md` 로드
    → 해당 도메인의 보정 IRR 확인 (신뢰도 A·B등급만 적용)
  → **ZM_raw > 2.0 시** → `references/zm-correction-model.md` 로드
    → ZM_eff = 2.0 + 0.68 × ln(ZM_raw / 2.0) 산출 → ⑥에 ZM_eff 입력
    → ZM_raw ≤ 2.0이면 기존 ZM 그대로 적용 (보정 불필요)

### Step 4: 어댑터가 스크리닝을 조정하는 방식

어댑터 세팅 후, §2 스크리닝 결과를 다음과 같이 **재해석**:

| 조정 대상 | 방법 |
|----------|------|
| L2 "적용됨/약함/부재" 판정 | 킬러 공식에 해당하는 공식이 "약함/부재"이면 **치명적 약점**으로 승격 |
| V 체크 | 어댑터 V2 등급(절대/강/중간/무관/사망)에 따라 V2 위반의 심각도 재설정 |
| L3 활성/비활성 판정 | 어댑터 L3 계층 1순위가 "비활성"이면 **전파 병목**으로 승격 |

## §3. 스포크 라우팅 테이블

스크리닝 결과 "약함/부재"인 항목에 대해서만 상세 스포크를 로드.

| 스크리닝 결과 | 로드할 스포크 |
|---|---|
| L1 축 이해 필요 | `references/layer1-mechanism.md` |
| L2 공식 이해 필요 | `references/layer2-formulas.md` |
| L3 전파 이해 필요 | `references/layer3-propagation.md` |
| 공식① 약함/부재 | `references/rx-absence.md` |
| 공식② 약함/부재 | `references/rx-void.md` |
| 공식③ 약함/부재 | `references/rx-reference.md` |
| 공식④ 약함/부재 | `references/rx-emotion.md` |
| 공식⑤ 약함/부재 | `references/rx-peak.md` |
| 공식⑥ 약함/부재 | `references/rx-repetition.md` |
| 도메인 특수 고려 | `references/domain-universal.md` |
| 세부도메인 어댑터 | `references/subdomain-adapters.md` (§2.5에서 판별된 세부도메인 섹션만) |
| V3-t 감쇠 예측 필요 | `references/v3t-decay-model.md` (반복 노출·장기 운영 관련 시) |
| V3-t 복원 전략 필요 | `references/v3t-decay-model.md` §V3 복원 곡선 모델 (대응 전략 정량화) |
| 하이브리드 도메인 판별 (부모) | `references/cross-domain-matrix.md` (2개+ 도메인 해당 시, §2.5 Step 5) |
| 하이브리드 세부도메인 매칭 | `references/subdomain-full-matrix.md` (세부도메인 특정 가능 시, 528쌍 전수) |
| L3 딜레이 분석 필요 | `references/l2l3-delay-model.md` (전파 속도·병목 분석 시) |
| 트리플 하이브리드 판별 | `references/triple-hybrid-matrix.md` (3개+ 도메인 해당 시, 5,456 조합) |
| E 전파 가속 판별 | `references/e-propagation-model.md` (E 채널 전략 시 PAM·IRR·NPE·BFC·DAC 정량 판정) |
| 팬덤 가속 판별 | `references/fandom-acceleration.md` (L3 축A 팬덤 재생산 활성 시 FAM·NPE_fandom 정량 판정) |
| 다채널 전파 전략 | `references/multi-channel-propagation.md` (E 채널 조합 최적화 시 CSE·MCR·MC-DAC 판정) |
| IRR 보정 판별 | `references/irr-calibration.md` (E 전파 시 IRR 파라미터 정밀 보정이 필요할 때 역산 파이프라인 로드) |
| ZM 포화 보정 | `references/zm-correction-model.md` (ZM_raw > 2.0 시 포화 보정 적용. ZM≤2.0은 기존 모델 유지) |

**원칙**: 필요한 스포크만 로드. 6개 처방카드를 한 번에 전부 로드하지 않는다. 세부도메인 어댑터도 해당 1개만 로드.
