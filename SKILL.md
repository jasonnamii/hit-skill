# Hit Skill — 히트 패턴 아키텍처 변환 엔진

인간이 왜 반응하는지(L1), 무엇을 어떻게 던져야 하는지(L2), 왜 퍼지는지(L3)의 3층 프레임워크를 적용하여 작업물·고민·초안을 변환하는 실행 도구.

## §1. 모드 자동 판별

| 입력 유형 | 모드 | 판별 기준 |
|---|---|---|
| 완성된 작업물 | **진단** | 문서·카피·기획서·제품 등 구체적 산출물이 존재 |
| 목표·고민만 | **설계** | 산출물 없이 "이걸 히트시키려면?" "어떻게 만들어야?" |
| 초안 + 의도 | **변환** | 초안이 있고 "이걸 히트 관점에서 바꿔줘" 요청 |

**애매한 경우**: 형에게 "진단/설계/변환 중 어느 모드로 진행할까요?" 확인. 복합 시 진단→설계→변환 순서 파이프라인 가능.

---

## §2. 1차 스크리닝 (3모드 공통)

입력물을 3층 프레임워크로 빠르게 스캔. **각 항목 = 1줄 판정. 전수 분석이 아님.**

> **⚠️ 근거 인용 필수**: 각 축/공식의 활성·비활성 판정 시, 입력물의 **구체적 단어·구절·맥락**을 직접 인용하여 근거를 제시하라. 일반론("이 도메인에서는 보통~")이나 인용 없는 판정 금지. 예: "'무료 체험 14일'이 축A(호혜)를 활성화 — 선행 호의 구조".

> **L3 우선 분기**: 범용 도메인(정치·공공·교육·투자자설득 등 6대 도메인 밖)에서는 L3(전파구조)가 L2(자극설계)를 압도하는 경우가 빈번. 범용 도메인 진입 시 **L3 5축을 먼저 스캔** → L3 축이 2개+ 강활성이면 **L3→L2→L1 역순 파이프라인** 허용. 이유: 정치 슬로건·공공 캠페인 등은 "어떻게 퍼지는가"가 "무엇을 던지는가"보다 성패를 결정하므로, 전파 구조를 먼저 확정한 뒤 자극을 그 구조에 맞게 설계하는 것이 효율적. 상세는 `references/domain-universal.md` §범용 도메인 L3 우선 스크리닝 참조.

→ **스캔 체크 테이블(L1 16축·L2 6공식·V 전제조건·L3 5축): `references/screening-tables.md` 로드**

본 스캔은 빠르게 훑어 "활성/잠재/비활성" 1줄 판정만 확정하는 단계. 각 축·공식의 상세 정의·적용례는 로드되는 스포크(L1·L2·L3·rx-*)에서 다룬다.

## §2.5. 도메인 어댑터 라우팅

### Python 라우터 우선 (토큰 절약)

```bash
# 도메인+세부도메인 판별
python scripts/domain_router.py "게임" "VR"

# 6대 도메인+33세부도메인 목록
python scripts/domain_router.py --list

# 접경지대 14건 판별 기준
python scripts/domain_router.py --border

# 스크리닝 결과→로드할 스포크 매핑
python scripts/domain_router.py --spokes L1 공식① E전파
```

→ `references/domain-routing.md` 로드 (6대 도메인 판별 → 33개 세부도메인 → 어댑터 파라미터 세팅 → 하이브리드/트리플 판별 → E 전파 가속 → 스크리닝 재해석)

**출력 필수 — 도메인 어댑터 판별 결과를 반드시 리포트에 기재하라:**

## §4. 모드별 실행

**3모드 공통 ⓪**: 도메인·세부도메인 판별 + 어댑터 세팅 (§2.5)

**3모드 공통 파이프라인 (§2→§2.5→§3→§4.5):**
①1차 스크리닝(§2) → ②도메인 어댑터(§2.5) → ③필요 스포크만 로드(§3) → ④모드별 분석(아래) → ⑤리포트 변환(§4.5)

| 모드 | ④ 모드별 분석 (공통 파이프 ①~③ 이후) | 필수 스포크 최소셋 |
|------|----------|----------|
| **진단** | 강점 Top 3 → 약점(킬러 우선) → 처방카드 → V 위반 경고 → L3 취약점 | L1 활성축 스포크 + 킬러공식 rx + domain-routing |
| **설계** | 목표·타겟 → L1 공략 2~3개 → L2 공식+V체크 → L3 전파+시대정신 → 프리모르템 → 액션 | L1 공략축 스포크 + 선택 L2 rx + L3 해당축 |
| **변환** | 변환 포인트 → 처방카드 적용 → 리라이팅 → Before/After → V 체크 | 활성 rx만 (비활성 rx 로드 금지) |
